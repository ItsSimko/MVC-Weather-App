Recently we spent a good deal of time supporting Type\+Script to enable us to create the Angular Font Awesome component. During that adventure we \href{https://basarat.gitbooks.io/typescript/docs/tips/defaultIsBad.html}{\texttt{ were}} \href{https://blog.neufund.org/why-we-have-banned-default-exports-and-you-should-do-the-same-d51fdc2cf2ad}{\texttt{ convinced}} that we were going to remove default exports from all of our components, libraries, and packages. This is complete with the umbrella release of {\ttfamily 5.\+1.\+0} of Font Awesome.

What does that mean?

\sout{Old way\+:}


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ fontawesome\ from\ '@fortawesome/fontawesome'}
\DoxyCodeLine{import\ solid\ from\ '@fortawesome/fontawesome-\/free-\/solid'}
\DoxyCodeLine{import\ faTwitter\ from\ '@fortawesome/fontawesome-\/free-\/brands/faTwitter'}
\DoxyCodeLine{import\ FontAwesomeIcon\ from\ '@fortawesome/vue-\/fontawesome'}
\DoxyCodeLine{}
\DoxyCodeLine{fontawesome.library.add(solid,\ faTwitter)}

\end{DoxyCode}


New way\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ \{\ library\ \}\ from\ '@fortawesome/fontawesome-\/svg-\/core'}
\DoxyCodeLine{import\ \{\ fas\ \}\ from\ '@fortawesome/free-\/solid-\/svg-\/icons'}
\DoxyCodeLine{import\ \{\ faTwitter\ \}\ from\ '@fortawesome/free-\/brands-\/svg-\/icons'}
\DoxyCodeLine{import\ \{\ FontAwesomeIcon\ \}\ from\ '@fortawesome/vue-\/fontawesome'}
\DoxyCodeLine{}
\DoxyCodeLine{library.add(fas,\ faTwitter)}

\end{DoxyCode}

\begin{DoxyCode}{0}

\end{DoxyCode}

Tree shaking is now functional by default and no additional configuration is required to make it work.

The {\ttfamily shakable.\+es.\+js} module has been removed and is no longer needed.

If you\textquotesingle{}ve previously configured tree shaking by modifying your webpack or rollup you can safely remove these.

{\bfseries{We recommend that you check your bundle size after upgrading an ensure that file sizes are as you would expect.}}
