Inside the {\ttfamily enter} function, calling {\ttfamily this.\+skip()} will prevent the node\textquotesingle{}s children being walked, or the {\ttfamily leave} function (which is optional) being called.

Call {\ttfamily this.\+replace(new\+\_\+node)} in either {\ttfamily enter} or {\ttfamily leave} to replace the current node with a new one.

Call {\ttfamily this.\+remove()} in either {\ttfamily enter} or {\ttfamily leave} to remove the current node.\hypertarget{md__c_1_2_users_2alexs_2source_2repos_2_s_q_8_term_project_8my_weather_2_s_q_8_term_project_8my_cf2980a28e6a3d07dbe4585cf2b5c81d_autotoc_md2733}{}\doxysection{\texorpdfstring{Why not use estraverse?}{Why not use estraverse?}}\label{md__c_1_2_users_2alexs_2source_2repos_2_s_q_8_term_project_8my_weather_2_s_q_8_term_project_8my_cf2980a28e6a3d07dbe4585cf2b5c81d_autotoc_md2733}
The ESTree spec is evolving to accommodate ES6/7. I\textquotesingle{}ve had a couple of experiences where \href{https://github.com/estools/estraverse}{\texttt{ estraverse}} was unable to handle an AST generated by recent versions of acorn, because it hard-\/codes visitor keys.

estree-\/walker, by contrast, simply enumerates a node\textquotesingle{}s properties to find child nodes (and child lists of nodes), and is therefore resistant to spec changes. It\textquotesingle{}s also much smaller. (The performance, if you\textquotesingle{}re wondering, is basically identical.)

None of which should be taken as criticism of estraverse, which has more features and has been battle-\/tested in many more situations, and for which I\textquotesingle{}m very grateful.\hypertarget{md__c_1_2_users_2alexs_2source_2repos_2_s_q_8_term_project_8my_weather_2_s_q_8_term_project_8my_cf2980a28e6a3d07dbe4585cf2b5c81d_autotoc_md2734}{}\doxysection{\texorpdfstring{License}{License}}\label{md__c_1_2_users_2alexs_2source_2repos_2_s_q_8_term_project_8my_weather_2_s_q_8_term_project_8my_cf2980a28e6a3d07dbe4585cf2b5c81d_autotoc_md2734}
