\begin{DoxyItemize}
\item {\bfseries{BREAKING}}\+: {\ttfamily jwt.\+verify} now requires an {\ttfamily algorithm} parameter, and {\ttfamily jws.\+create\+Verify} requires an {\ttfamily algorithm} option. The {\ttfamily "{}alg"{}} field signature headers is ignored. This mitigates a critical security flaw in the library which would allow an attacker to generate signatures with arbitrary contents that would be accepted by {\ttfamily jwt.\+verify}. See \href{https://auth0.com/blog/2015/03/31/critical-vulnerabilities-in-json-web-token-libraries/}{\texttt{ https\+://auth0.\+com/blog/2015/03/31/critical-\/vulnerabilities-\/in-\/json-\/web-\/token-\/libraries/}} for details.
