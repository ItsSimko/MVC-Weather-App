Finds the common standard cache directory \end{quote}
The \href{https://github.com/istanbuljs/nyc}{\texttt{ {\ttfamily nyc}}} and \href{https://ava.li}{\texttt{ {\ttfamily AVA}}} projects decided to standardize on a common directory structure for storing cache information\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\#\ nyc}
\DoxyCodeLine{./node\_modules/.cache/nyc}
\DoxyCodeLine{}
\DoxyCodeLine{\#\ ava}
\DoxyCodeLine{./node\_modules/.cache/ava}
\DoxyCodeLine{}
\DoxyCodeLine{\#\ your-\/module}
\DoxyCodeLine{./node\_modules/.cache/your-\/module}

\end{DoxyCode}


This module makes it easy to correctly locate the cache directory according to this shared spec. If this pattern becomes ubiquitous, clearing the cache for multiple dependencies becomes easy and consistent\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{rm\ -\/rf\ ./node\_modules/.cache}

\end{DoxyCode}

Finds the cache directory using the supplied options. The algorithm checks for the {\ttfamily CACHE\+\_\+\+DIR} environmental variable and uses it if it is not set to {\ttfamily true}, {\ttfamily false}, {\ttfamily 1} or {\ttfamily 0}. If one is not found, it tries to find a {\ttfamily package.\+json} file, searching every parent directory of the {\ttfamily cwd} specified (or implied from other options). It returns a {\ttfamily string} containing the absolute path to the cache directory, or {\ttfamily undefined} if {\ttfamily package.\+json} was never found or if the {\ttfamily node\+\_\+modules} directory is unwritable.\hypertarget{md__c_1_2_users_2alexs_2source_2repos_2_s_q_8_term_project_8my_weather_2_s_q_8_term_project_8my_ae0141d2b9c645a0f91d40b0b27589d1_autotoc_md2978}{}\doxysubsubsection{\texorpdfstring{options}{options}}\label{md__c_1_2_users_2alexs_2source_2repos_2_s_q_8_term_project_8my_weather_2_s_q_8_term_project_8my_ae0141d2b9c645a0f91d40b0b27589d1_autotoc_md2978}
