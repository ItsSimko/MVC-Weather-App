\begin{DoxyEnumerate}
\item Rollup and esbuild do not support using {\ttfamily enforce} to control the order of plugins. Users need to maintain the order manually.
\item Webpack\textquotesingle{}s id filter is outside of loader logic; an additional hook is needed for better perf on Webpack. In Rollup and Vite, this hook has been polyfilled to match the behaviors. See for the following usage examples.
\item Although esbuild can handle both Java\+Script and CSS and many other file formats, you can only return Java\+Script in {\ttfamily load} and {\ttfamily transform} results.
\item Currently, {\ttfamily write\+Bundle} is only serves as a hook for the timing. It doesn\textquotesingle{}t pass any arguments.
\end{DoxyEnumerate}
