Be sure to read the \href{https://eslint.org/docs/developer-guide/working-with-rules}{\texttt{ official ESLint guide}} before you start writing a new rule.

To see what an abstract syntax tree (AST) of your code looks like, you may use \href{https://astexplorer.net}{\texttt{ AST Explorer}}. After opening \href{https://astexplorer.net}{\texttt{ AST Explorer}}, select {\ttfamily Vue} as the syntax and {\ttfamily vue-\/eslint-\/parser} as the parser.

The default Java\+Script parser must be replaced because \href{https://vuejs.org/guide/scaling-up/sfc.html}{\texttt{ Vue.\+js single file components}} are not plain Java\+Script, but a custom file format. \href{https://github.com/vuejs/vue-eslint-parser}{\texttt{ {\ttfamily vue-\/eslint-\/parser}}} is a replacement parser that generates an enhanced AST with nodes that represent specific parts of the template syntax, as well as the contents of the {\ttfamily \texorpdfstring{$<$}{<}script\texorpdfstring{$>$}{>}} tag.

To learn more about certain nodes in a produced AST, see the \href{https://github.com/estree/estree}{\texttt{ ESTree project page}} and the \href{https://github.com/vuejs/vue-eslint-parser/blob/master/docs/ast.md}{\texttt{ vue-\/eslint-\/parser AST documentation}}.

{\ttfamily vue-\/eslint-\/parser} provides a few useful parser services to help traverse the produced AST and access template tokens\+:


\begin{DoxyItemize}
\item {\ttfamily context.\+parser\+Services.\+define\+Template\+Body\+Visitor(visitor, script\+Visitor)}
\item {\ttfamily context.\+parser\+Services.\+get\+Template\+Body\+Token\+Store()}
\end{DoxyItemize}

Check out an \href{https://github.com/vuejs/eslint-plugin-vue/blob/master/lib/rules/mustache-interpolation-spacing.js}{\texttt{ example rule}} to see usage of these services.

Be aware that depending on the code samples you write in tests, the {\ttfamily Rule\+Tester} parser property must be set accordingly (this can be done on a test by test basis). See an \href{https://github.com/vuejs/eslint-plugin-vue/blob/master/tests/lib/rules/attribute-hyphenation.js\#L19}{\texttt{ example here}}.
